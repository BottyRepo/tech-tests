## Питання 1 — JavaScript / Event Loop

Є два фрагменти коду. Кожна функція виконує **математичні обчислення** (без читання з диску, без мережевих запитів) і займає **~10 секунд** виконання.

```js
function heavyCalc1() {
  return new Promise((resolve) => {
    let result = 0;
    const end = Date.now() + 10_000;
    while (Date.now() < end) {
      result += Math.sqrt(Math.random());
    }
    resolve(result);
  });
}

function heavyCalc2() {
  return new Promise((resolve) => {
    let result = 0;
    const end = Date.now() + 10_000; 
    while (Date.now() < end) {
      result += Math.sqrt(Math.random());
    }
    resolve(result);
  });
}

const [r1, r2] = await Promise.all([heavyCalc1(), heavyCalc2()]);
```

### Питання до кандидата:

1. Скільки секунд займе виконання `Promise.all` — **~10 сек чи ~20 сек**? Чому?
2. Чи виконуються ці дві функції **паралельно**? Поясни механізм.
3. Що таке **event loop** і як він пов'язаний з цією поведінкою?
4. Які методи оптимізації **швидкості** ти б застосував до цього коду?
